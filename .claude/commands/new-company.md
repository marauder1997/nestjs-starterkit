# /new-company — 새 기업 커스터마이징

새 지원 기업의 정보를 입력받아 `src/data/` 아래 JSON 파일 3개를 한 번에 업데이트합니다.

## 실행 순서

### 1단계 — 기업 정보 인터뷰 (한 번에 하나씩 질문)

아래 항목을 순서대로 질문한다. 답변이 없으면 기본값을 사용한다.

| 항목 | 질문 | 기본값 |
|------|------|--------|
| 회사명(한글) | 회사명을 한글로 입력해 주세요. | 글로벌파트너스 |
| 회사명(영문) | 회사명을 영문으로 입력해 주세요. | Global Partners Co. |
| 슬로건 | 회사 슬로건(한 문장)을 입력해 주세요. | 세계로 나아가는 당신의 든든한 파트너 |
| 설명 | 회사 소개 한 줄을 입력해 주세요. | 해외진출 전문 컨설팅 기업 |
| 설립연도 | 설립연도를 입력해 주세요. (예: 2010) | 2010 |
| 직원수 | 직원 수를 입력해 주세요. (예: 50+) | 50+ |
| 주소 | 회사 주소를 입력해 주세요. | 서울특별시 강남구 |
| 전화 | 대표 전화번호를 입력해 주세요. | 02-0000-0000 |
| 이메일 | 대표 이메일을 입력해 주세요. | contact@company.kr |
| 진출국가수 | 지원 가능한 국가 수를 입력해 주세요. | 10 |
| 프로젝트수 | 지금까지 지원한 프로젝트 수를 입력해 주세요. | 50+ |

### 2단계 — 서비스 항목 확인

"기본 서비스 6개를 그대로 사용하시겠어요? (Y/n)" 를 묻는다.
- Y → `src/data/services.json` 유지
- N → 어떤 서비스로 변경할지 물어보고 수정

### 3단계 — 포트폴리오 초기화 여부 확인

"포트폴리오를 비우고 새로 시작하시겠어요? (Y/n)" 를 묻는다.
- Y → `src/data/portfolio.json`을 빈 배열 `[]`로 초기화
- N → 기존 샘플 데이터 유지

### 4단계 — JSON 파일 업데이트

수집한 정보로 `src/data/company.json`을 업데이트한다.
- 기존 `history`, `partners`, `social` 필드는 유지
- 입력받은 값만 덮어쓴다

### 5단계 — 완료 보고

아래 형식으로 결과를 출력한다.

```
✅ 커스터마이징 완료

회사명: [입력값]
수정 파일:
  - src/data/company.json  ← 회사 정보 업데이트
  - src/data/portfolio.json ← (초기화한 경우)

다음 단계:
  1. src/data/company.json 에서 연혁(history) 수정
  2. src/data/portfolio.json 에 실제 프로젝트 추가 (/add-portfolio 사용)
  3. pnpm dev 로 결과 확인 → http://localhost:3000
```

## 주의사항
- `src/data/` 외 파일은 절대 수정하지 않는다
- 기존 JSON 파일을 덮어쓰기 전에 변경 내용을 먼저 보여주고 확인을 받는다
- 확인 없이 파일을 삭제하지 않는다
